# CVdoku — Real-Time AR Sudoku Solver

> Point your camera at any Sudoku puzzle. CVdoku detects the grid, recognises the digits, solves the puzzle instantly, and projects the solution back onto the live feed in augmented reality.

![Demo GIF coming soon](resources/demo.gif)

---

## Features

- Real-time grid detection with perspective correction
- Custom-trained CNN digit classifier (trained from scratch on MNIST + augmentation)
- Confidence thresholding — blank cells are never misclassified
- Frame stability voting — majority vote across multiple frames before locking solution
- AR overlay — solved digits projected back onto the puzzle in the live camera feed
- Fast backtracking solver
- Configurable via command-line arguments

## Setup

### 1. Clone the repo
```bash
git clone https://github.com/parigh1/CVdoku.git
cd CVdoku
```

### 2. Create a virtual environment
```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Train the digit model
```bash
python train.py
# Trains a CNN on MNIST, saves to models/digit_model.h5
# Takes ~2-3 minutes on CPU
```

### 5. Run CVdoku
```bash
python main.py
# Options:
#   --camera   Camera index (default: 0)
#   --model    Path to model file (default: models/digit_model.h5)
#   --width    Frame width (default: 1280)
#   --height   Frame height (default: 720)
```

## How it works

1. **Grid detection** — Canny edge detection + contour finding locates the largest quadrilateral in frame. Perspective warp squares it up.
2. **Digit recognition** — The warped grid is split into 81 cells. Each cell is fed to a CNN trained on MNIST digits. Cells with low confidence are treated as blank.
3. **Frame stability** — Classification runs across multiple consecutive frames; a majority vote per cell prevents jitter from causing wrong reads.
4. **Solving** — A backtracking algorithm solves the 9×9 grid in milliseconds.
5. **AR overlay** — The solution digits are rendered onto a blank canvas, then inverse-warped back into the original camera perspective and composited onto the live feed.
