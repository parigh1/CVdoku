# CVdoku — Real-Time AR Sudoku Solver

> Point your camera at any Sudoku puzzle. CVdoku detects the grid, recognises the digits, solves the puzzle, and projects the solution back onto the live feed in augmented reality.

## Features

- **Real-time grid detection** — Canny edge detection + contour finding with perspective warp
- **Custom-trained CNN** — trained from scratch on MNIST + printed-digit augmentation (99.67% val accuracy)
- **Confidence thresholding** — blank cells detected by pixel density, never misclassified
- **Geometric post-processing** — horizontal bar detection disambiguates `1` vs `7`
- **Frame stability voting** — majority vote across 7 frames before locking solution
- **Conflict recovery** — automatic digit-swap recovery for common CNN misreads
- **AR overlay** — solved digits inverse-warped back into the camera perspective
- **Configurable** — camera index, model path, resolution, threshold all via argparse

## Setup

### 1. Clone
```bash
git clone https://github.com/parigh1/CVdoku.git
cd CVdoku
```

### 2. Virtual environment
```bash
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Train the digit model
```bash
python train.py
# Downloads MNIST automatically (~11MB)
# Trains CNN with printed-digit augmentation
# Saves to models/digit_model.h5
# Takes ~5 min on CPU, ~1 min on GPU
```

### 5. Run
```bash
python main.py

# Options:
python main.py --camera 1          # different camera index
python main.py --threshold 0.90    # stricter digit confidence
python main.py --stability-frames 5  # fewer frames before solving
python main.py --width 1280 --height 720
```

## How It Works

```
Camera Frame
     │
     ▼
VisionEngine.pre_process()        Grayscale → Gaussian blur → Adaptive threshold
     │
     ▼
VisionEngine.find_board_contour() Largest quadrilateral contour = Sudoku grid
     │
     ▼
VisionEngine.get_warp()           Perspective transform → 630×630 flat board
     │
     ▼
VisionEngine.split_boxes()        Slice into 81 cells (70×70px each)
     │
     ▼
FrameVoter (7 frames)             Majority vote per cell for stability
     │
     ▼
DigitClassifier.predict()         CNN → softmax → confidence threshold
     │                            + geometric 1 vs 7 disambiguation
     ▼
SudokuSolver.solve_with_recovery() Backtracking + digit-swap conflict recovery
     │
     ▼
OverlayEngine.draw()              Render digits → inverse warp → composite on frame
```

## CNN Architecture

```
Input (28×28×1)
├── Conv2D(32) → BN → Conv2D(32) → MaxPool → Dropout(0.25)
├── Conv2D(64) → BN → Conv2D(64) → MaxPool → Dropout(0.25)
├── Conv2D(128) → BN → Dropout(0.25)
└── Dense(256) → BN → Dropout(0.5) → Dense(9, softmax)

Training data : MNIST (1-9 only) + printed-digit augmentation
                (eroded, dilated, binary variants)
Val accuracy  : 99.67%
Model size    : ~1.8 MB
```

## Known Limitations & Roadmap

### Current limitation — printed font digit recognition

The CNN is trained on MNIST (handwritten digits) with synthetic augmentation. This works well for most digits but struggles with specific app fonts where `1` closely resembles `7` in terms of pixel distribution after binarization.

**What was tried:**
- Confidence thresholding — improved blank cell detection significantly
- Aspect ratio disambiguation — unreliable due to serif fonts affecting width measurement  
- Horizontal bar detection — geometrically sound but the app's `1` has a subtle top serif that triggers false positives
- Printed-digit augmentation (erosion, dilation, binary variants of MNIST) — improved `5/6` confusion, did not resolve `1/7`

**Root cause:** MNIST `1`s are handwritten loops and strokes. The target app uses a sans-serif font where `1` is a thin vertical stroke with a small angled top stroke — visually distinct to humans, but producing similar pixel statistics to `7` after adaptive thresholding.

**Planned fix — v2 retrain on printed digit datasets:**

| Dataset | Source | Why |
|---|---|---|
| `pintowar/numerical-images` | Kaggle | MNIST + Chars74K combined, best coverage |
| `supreethrao/chars74kdigitalenglishfont` | Kaggle | Pure digital font digits, multiple typefaces |
| `kshitijdhama/printed-digits-dataset` | Kaggle | Printed digit images specifically |

Expected improvement: **~90% chance of eliminating the `1/7` confusion** — Chars74K contains clean sans-serif printed digits across multiple fonts that directly match Sudoku app rendering.

Hardware for retrain: RTX 3050 (CUDA) — estimated **under 10 minutes**.
